GitFlow工作记录

发表于 2018-10-10 | 分类于 Git

本文用于记录 使用Git进行团队协作开发的工作流程，不涉及Git安装和配置

GitFlow工作流：

主要分支及功能：

master(线上版本)

  • 用于产品的功能全部实现后，在master分支对外发布，一般此分支只能从其他分支合并，不能直接修改

develop:(主开发分支)

  • 包含所有要发布到下一个master的代码，这个主要合并于其他分支，比如feature分支

feature

  • 这个分支主要是用来开发一个新的功能，一旦开发完成，我们合并回Develop分支进入下一个Release

release

*　当你需要一个发布一个新Release的时候，我们基于Develop分支创建一个Release分支，完成Release后，我们合并到Master和Develop分支

otfix 当我们在Production发现新的Bug时候，我们需要创建一个Hotfix, 完成Hotfix后，我们合并回Master和Develop分支，所以Hotfix的改动会进入下一个Release.
